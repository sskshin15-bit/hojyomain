# 補助金データベース（subsidiesテーブル）の使い方ガイド

このファイルは、

- プログラミングが分からない人
- 中学生くらいの人

でも「最新の補助金データ」を正しく更新できるようにするための説明書です。

コードを触る必要はありません。  
**Supabase（スーパーベース）の画面だけを操作すればOK**です。

---

## 1. 何を管理しているテーブルなの？

`subsidies` テーブルは、このサービスで使う

- 「最新の補助金・助成金」の一覧

を保存している場所です。

AI は、ここに入っているデータだけを見て、

- 顧問先の会社に合いそうな補助金を
- 最大3つまで

選んでくれます。

**ここに入っていない補助金は、AIは提案してはいけないルール**になっています。

---

## 2. どこから編集するの？

1. ブラウザで Supabase のダッシュボードを開きます。
2. 対象プロジェクトを選びます。
3. 左メニューから **「Table editor（テーブルエディタ）」** を開きます。
4. テーブル一覧から **`public.subsidies`** をクリックします。

これで、「補助金一覧の表」が見えるはずです。

---

## 3. カラム（列）の意味

`subsidies` テーブルには、こんな列があります。

- **id**  
  - 自動で振られるIDです。  
  - **触らないでください。**

- **name**（必須）  
  - 補助金の名前です。  
  - 例: `IT導入補助金2026（通常枠）`

- **agency**  
  - どこの省庁・機関の補助金か。  
  - 例: `経済産業省`, `日本商工会議所`

- **target_industries**  
  - 対象となる業種のリストです。  
  - JSON 形式（カッコとダブルクォーテーション）で書きます。  
  - 例:
    - `["全業種"]`
    - `["製造業", "建設業", "情報通信業"]`
    - `["飲食業", "小売業", "サービス業"]`

- **max_amount**  
  - 上限金額です（数字）。  
  - 単位は「円」です。  
  - 例: `1500000`（= 150万円）

- **description**  
  - この補助金は何を目的にしているか、どんなことに使えるか。  
  - 短めの説明文を日本語で書きます。

- **url**  
  - 公式の公募要領ページなどのリンクです。  
  - 無くても動きますが、入れておくと便利です。

- **deadline**  
  - 申請の締切日時です。  
  - いつまで募集しているかを書きます。  
  - 例: `2026-03-31 23:59:59+09` など。

- **is_active**  
  - この補助金が「今現在、公募中かどうか」です。  
  - `true` → 今も使える（AIの候補に入る）  
  - `false` → 受付終了（AIの候補から外れる）

- **created_at**  
  - いつ登録されたかの日時です。  
  - システムが自動で入れるので、**基本的に触らなくてOK**です。

---

## 4. 「編集していい列」と「編集してはいけない列」

### 編集していい列

- `name`
- `agency`
- `target_industries`
- `max_amount`
- `description`
- `url`
- `deadline`
- `is_active`

### 基本的に触らないほうがいい列

- `id`
- `created_at`

これらはシステム側で管理しているので、消したり書き換えたりすると、  
あとで原因が分からなくなりやすいです。

---

## 5. 新しい補助金を「追加」する方法

1. Supabase の `Table editor` で `public.subsidies` を開きます。
2. 右上の **「+ Insert row」** または **「New row」** ボタンを押します。
3. 追加画面が出るので、次のように入力します。

   - `name`: 補助金の正式名称
   - `agency`: 省庁・機関名（分からなければ空でもOK）
   - `target_industries`:
     - 例: `["全業種"]` や `["製造業", "建設業"]`
   - `max_amount`:
     - 例: `1500000`（150万円）
   - `description`:
     - どんな目的の補助金か、1〜3行くらいで。
   - `url`:
     - 公式ページのURL（あれば）
   - `deadline`:
     - 締切が分かっていれば入力。未定なら空でもOKですが、分かれば入れたほうが良いです。
   - `is_active`:
     - 今募集しているなら `true`
     - まだ発表前・既に締切後であれば `false`

4. 入力が終わったら **Save（保存）** します。

これで、AI は次回の判定から、  
この新しい補助金を候補として使えるようになります。

---

## 6. 受付が終わった補助金を「止める」方法

補助金の受付期間が終わったときは、レコードを消さずに、

- `is_active` を `false` に変える

だけでOKです。

### 手順

1. `public.subsidies` を開きます。
2. 該当する補助金の行を探します。
3. その行の `is_active` の値を `false` に変更します。
4. 保存します。

これで、その補助金は

- 過去の記録としては残る
- でも、**AIの提案候補には出てこなくなる**

という状態になります。

## 7. よくある質問（Q&A）

### Q1. 間違って max_amount の数字を変えてしまいました。どうなりますか？

AI が参照する「上限金額」のイメージが変わります。  
ただし、サービス全体が壊れることはありません。

心配な場合は、**正しい数字に戻す**か、わからなければ  
チームの詳しい人 or Cursor のAIに「この値はどれくらいが正しそうか」相談してください。

---

### Q2. target_industries を間違った書き方にしてしまいました。

例: `["製造業, 建設業"]` など。

この場合、その行の業種条件がうまく判定されず、  
「全業種向けに近い扱い」になることがあります。

サービス自体はエラーになりにくいですが、  
**できるだけ正しい JSON 形式**（`["製造業", "建設業"]`）で直してください。

---

### Q3. 行を削除しても大丈夫ですか？

技術的には削除できますが、

- 過去にその補助金で判定した履歴とのつながり

が分かりにくくなることがあります。

基本ルールとしては、

- **削除よりも `is_active = false` にする**

をおすすめします。

---

## 8. 困ったときの対処法

もし操作に自信がなかったり、よく分からなくなったときは、

1. Cursor を開きます。
2. このプロジェクトを選びます。
3. AI に、なるべく具体的に状況を書いて相談してください。

例:

> subsidiesテーブルで新しい補助金を追加したいのですが、どのカラムに何を書けばいいか分かりません。
> IT導入補助金2026の通常枠を追加したいです。

このように伝えると、AI がこのファイルの内容も参考にしながら、  
**画面でどこをどう触ればいいか**を丁寧に教えてくれます。

- 分からないときは、**無理にいじらず、一度 AI やチームに相談する**習慣をつけると安全です。
