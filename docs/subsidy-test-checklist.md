# 補助金機能 テストのやり方（やさしい版）

中学生でもわかるように、一つずつ丁寧に説明します。  
順番にやっていけば大丈夫です。

---

## テストを始める前に

まず、この3つができているか確認してください。

1. **プロジェクトを開いている**  
   Cursor で「hojyomain」というプロジェクトが開いていること。

2. **サーバーが動いている**  
   ターミナル（画面下の黒いところ）で `npm run dev` を実行して、「Ready」と出ていること。

3. **Supabase にログインできる**  
   ブラウザで Supabase のサイトを開いて、自分のプロジェクトに入れること。

できていなければ、まずはここまで進めてからテストを始めてください。

---

## テスト1: 補助金を1件登録してみる

**目的**: 補助金データがちゃんと登録されるか確かめます。

### やり方

1. **新しいターミナルを開く**  
   - 画面上のメニューから `Terminal` → `New Terminal` を選ぶ  
   - 別の黒い画面が開きます。

2. **次の命令を1行コピーする**（全部まとめてコピーでOKです）:

   ```
   curl -X POST http://localhost:3000/api/cron/sync-subsidies -H "Content-Type: application/json" -H "Authorization: Bearer my-super-secret-cron-key-reina0422" -d '{"subsidies":[{"api_id":"test-001","name":"テスト補助金","agency":"経済産業省","target_industries":["全業種"],"max_amount":1000000,"description":"テスト","is_active":true}]}'
   ```

3. **ターミナルに貼り付けて Enter キーを押す**

4. **結果を確認する**  
   - `{"message":"subsidies の同期が完了しました。","count":1}` と出ていれば **OK** です。  
   - 別のエラーが表示されたら、その内容をメモしておいてください。

5. **Supabase で確認する**  
   - ブラウザで Supabase を開く  
   - 自分のプロジェクトを選ぶ  
   - 左のメニューから「Table editor」をクリック  
   - テーブル一覧から「subsidies」をクリック  
   - 表に「テスト補助金」という行が増えていれば成功です。

---

## テスト2: AI が補助金を出してくれるか確かめる

**目的**: 画面から会社情報を入れたとき、AI が補助金を提案してくれるか確かめます。

### やり方

1. **ブラウザでサイトを開く**  
   アドレス欄に `http://localhost:3000` と入力して Enter。

2. **ログインする**  
   - すでにアカウントがあればログイン。  
   - なければ「サインアップ」で新規登録してからログイン。

3. **スクリーニング画面に行く**  
   ログインすると自動で「AI 5秒スクリーニング」の画面に飛ぶことが多いです。  
   飛ばなかった場合は、メニューなどから「スクリーニング」を選んでください。

4. **会社の情報を入力する**（なんでもOKです）  
   例:
   - 会社名: テスト株式会社
   - 代表者名: 山田太郎
   - 業種: 製造業
   - 従業員数: 50人
   - 資本金: 1000万円

5. **「AIで5秒スクリーニング」のボタンを押す**

6. **結果を確認する**  
   - エラーが出ずに、補助金のカードが1枚以上表示されれば **OK** です。  
   - 「補助金データが登録されていません」と出たら、テスト1 ができていないか、Supabase の subsidies に `is_active = true` のデータがない可能性があります。

---

## テスト3: 結果がちゃんと保存されているか確かめる

**目的**: AI の提案結果が、後から見返せるように保存されているか確かめます。

### やり方

1. **テスト2 を1回実行しておく**  
   （会社名は覚えておいてください。例: テスト株式会社）

2. **Supabase を開く**  
   ブラウザで Supabase のサイト → 自分のプロジェクト。

3. **clients テーブルを開く**  
   - 左メニュー「Table editor」  
   - テーブル一覧から「clients」をクリック。

4. **結果を確認する**  
   - 一番下あたりに、さっき入力した会社名の行があるか見る。  
   - その行の「recommended_subsidies」に、補助金の情報が入っていれば **OK** です。

---

## うまくいかなかったとき

- 出たエラーメッセージを、そのままコピーする。  
- Cursor の AI に、「このエラーが出ました」と貼り付けて相談する。  
- 「環境変数が設定されていません」と出た場合は、  
  `docs/operations-playbook.md` の「ローカル開発時の起動」を確認する。

---

## まとめ

| テスト | やること | 成功の目安 |
|--------|----------|------------|
| 1 | ターミナルで curl を実行 | 「同期が完了しました」と出る＆Supabase にデータが増える |
| 2 | ブラウザで会社情報を入れて AI 実行 | 補助金カードが表示される |
| 3 | Supabase の clients を見る | 入力した会社名の行に補助金の結果が入っている |

分からないところがあれば、無理せず大人や Cursor の AI に聞いてください。
